# Security Policy

## Supported versions

Security fixes are applied to the latest `main` branch.

## Reporting a vulnerability

If you find a security issue, do not open a public issue. Use GitHub's private
security advisory flow for this repository if available, or contact the
maintainers through a private channel.

Include:

- A short summary of the issue
- The affected component or command
- Steps to reproduce
- Any logs or proof of concept material

## Safe disclosure

Please allow time to investigate and patch before publishing details publicly.
